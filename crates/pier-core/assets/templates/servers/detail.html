Promote agent → standalone pier-core

Exports a bundle with all services, domains, environment variables, backup schedules and global configuration currently associated with . Apply it on that server so it can run as a full Pier panel instead of a lightweight agent.

Heads up

  • Target server must run pier-core (same major version). If it only has pier-agent, install pier-core first.
  • Target database must be empty (no existing services). The import refuses to overwrite.
  • Users / sessions / secrets from this panel are not exported — create an admin on the new panel when it starts.

Automatic (recommended)

Core pushes the bundle to the agent, which downloads pier-core, imports the bundle, installs a systemd unit, stops itself and starts the new pier-core. You watch the log on the target.

Manual alternative

1. Download the bundle

2. Apply on target server

# On the target server (as root):
systemctl stop pier-agent
scp bundle-<name>.json root@target:/tmp/
PIER_DATA_DIR=/var/lib/pier pier --import-bundle /tmp/bundle-<name>.json
systemctl enable --now pier
